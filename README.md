# mdX

A lightweight, immersive Markdown editor built with [Tauri](https://tauri.app/) + [CodeMirror 6](https://codemirror.net/).

## Features

- **Hybrid rendering** — Markdown syntax marks (`#`, `*`, `` ` ``) are hidden when the cursor is away, and revealed when you move the cursor to that line. Feels like a rendered document, edits like raw text.
- **Local-first** — Reads and writes `.md` / `.markdown` / `.mdown` files directly on disk. No cloud, no account, no network required.
- **Auto-save** — 800ms debounce after any edit. Ctrl+S for manual save.
- **File association** — Double-click any Markdown file to open it in mdX (via Windows registry entry).
- **Dark mode** — Follows system preference automatically.
- **Fast startup** — ~65ms from HTML parse to editor ready. Total cold launch dominated by WebView2 process spawn (~600–1200ms on first boot after reboot; <500ms on warm starts).

## Quick Start

### Download

Grab the latest release from the [Releases](../../releases) page.

### Build from source

Prerequisites:

- [Rust](https://rustup.rs/) (stable, 1.95+)
- [Node.js](https://nodejs.org/) (for esbuild bundling)
- WebView2 Runtime (pre-installed on Windows 10/11)

```bash
# 1. Vendor CodeMirror dependencies (first time only)
cd public
python vendorize.py   # or manually set up vendor/ from esm.sh

# 2. Bundle the frontend
./bundle.cmd          # runs: npx esbuild app.js --bundle --format=iife --minify --outfile=app.bundle.js

# 3. Build the release binary
cd ../src-tauri
cargo build --release
```

The executable is at `src-tauri/target/release/mdx.exe`.

> **Note:** The `custom-protocol` feature is enabled by default on the `tauri` dependency so that `cargo build --release` loads embedded assets correctly. If you use `cargo tauri build` instead, the CLI handles this automatically.

### Development mode

```bash
cd public
# Start a static file server (any will do)
npx serve -l 8080 .

# In another terminal
cd src-tauri
cargo tauri dev
```

## Architecture

```
public/                  # Frontend (served by Tauri's custom protocol)
  index.html             # Shell HTML with boot-timing capture
  style.css              # Theme + CodeMirror overrides
  app.js                 # Editor logic (CodeMirror 6 + Tauri integration)
  markdown-slim.js       # Minimal markdown LanguageSupport (no code-block highlighting)
  app.bundle.js          # Bundled & minified (generated by bundle.cmd)
  vendor/                # CodeMirror 6 ES modules (from esm.sh)
  bundle.cmd             # Frontend build script
  vendorize.py           # Script to download vendor/ from esm.sh

src-tauri/               # Rust backend (Tauri 2)
  src/
    main.rs              # Entry point
    lib.rs               # Tauri commands: read_file, write_file, get_cli_file, set_window_title
  Cargo.toml
  tauri.conf.json
  capabilities/default.json
```

The `markdown-slim.js` module constructs a `LanguageSupport` directly from `@lezer/markdown` + `@codemirror/language`, bypassing `@codemirror/lang-markdown` which would pull in ~170KB of code-block language highlighting (JS/CSS/HTML parsers + autocomplete) that a pure Markdown editor doesn't need.
